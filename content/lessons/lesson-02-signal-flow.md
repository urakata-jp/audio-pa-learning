# 音の流れを理解する

## このレッスンで学ぶこと

- 音響トラブル対応の基本は「信号の流れ」を追うこと
- マイク、ミキサー、アンプ、スピーカーの順番
- PC音源、録音、別室、通訳送りの考え方
- 音が出ない時の切り分け方

## 導入会話

:::dialogue
初心者:  
「本番前に音が出なかったら、どこから見ればいいですか？」

講師:  
「焦って全部触るのではなく、音の入口から出口まで順番に見ます。信号の流れを追える人は、現場で強いです。」
:::

## 基本解説

基本的なPAの流れは次の通りです。

[diagram: basic-signal-chain]

```text
マイク → ミキサー → アンプ → スピーカー
```

パワードスピーカーを使う場合は、スピーカー内部にアンプが入っているため次のようになります。

```text
マイク → ミキサー → パワードスピーカー
```

PC音源の場合は、PCの音声出力が入力になります。

```text
PC → オーディオインターフェースまたは変換ケーブル → ミキサー → スピーカー
```

講演会や同時通訳の現場では、メインスピーカー以外にも複数の送り先があります。

[diagram: multi-output-routing]

```text
講演者マイク
  ↓
ミキサー
  ├→ メインスピーカー
  ├→ 録音PC
  ├→ 通訳機
  ├→ 通訳室モニター
  └→ 控室・別室
```

## 現場での使い方

音が出ない時は、以下の順番で確認します。

1. マイクやPCから音が出ているか
2. ミキサーの入力メーターが振れているか
3. チャンネルがミュートされていないか
4. フェーダーが上がっているか
5. MAINやAUXに送られているか
6. アンプやスピーカーの電源が入っているか
7. ケーブルが正しく接続されているか
8. 出力先を間違えていないか

:::warning
音が出ない時に、最初からゲインやアンプ音量を大きく上げるのは危険です。原因が解決した瞬間に大音量が出ることがあります。
:::

## 講演会の場合

講演者のマイクがメインスピーカーに出ているだけでは不十分です。録音、別室、通訳機、控室へ必要な音が送られているか確認します。

## 出し物・歌イベントの場合

PCのBGM、マイク、場合によっては楽器やSEが混ざります。どの音がどのタイミングで必要かを事前に整理しておきます。

## 屋外イベントの場合

スピーカーが複数ある場合、メインと補助スピーカーのどちらに何が送られているか確認します。ケーブル距離が長くなるため、断線や接触不良も起きやすくなります。

## まとめ

音響の現場では、複雑な操作よりも「音がどこから来て、どこへ行くか」を理解することが重要です。信号の流れを追えるようになると、音が出ない、録音だけ小さい、別室だけ聞こえないといったトラブルを落ち着いて切り分けられます。
