# 図解作成指示一覧

CodexではSVGまたはHTML/CSSで図解カードとして作成してください。文字化けや誤表記を避けるため、画像生成AIだけで端子名や信号名を描かないでください。

## pa-basic-flow

目的: PAの基本構造を説明する。

図:
```text
入力 → 調整 → 出力
マイク/PC → ミキサー → アンプ/スピーカー
```

デザイン:
- 横並びの3ステップ
- 入力=緑、調整=青、出力=オレンジ
- アイコン: マイク、ミキサー、スピーカー

## basic-signal-chain

目的: マイクからスピーカーまでの信号経路。

図:
```text
マイク → ミキサー → アンプ → スピーカー
```

補足:
- パワードスピーカー版も併記
```text
マイク → ミキサー → パワードスピーカー
```

## multi-output-routing

目的: 講演会で音を複数の場所へ送る構造。

図:
```text
講演者マイク
  ↓
ミキサー
  ├→ メインスピーカー
  ├→ 録音PC
  ├→ 通訳機
  ├→ 通訳室モニター
  └→ 控室・別室
```

## inverse-square-law

目的: 距離が2倍になると約6dB下がることを視覚化。

図:
```text
1m: 100dB
2m: 94dB
4m: 88dB
8m: 82dB
16m: 76dB
```

デザイン:
- スピーカーから同心円状に距離表示
- 各距離にdB表示
- 「目安」「自由音場の場合」と注記

## mic-polar-patterns

目的: 指向性の違い。

図:
- 無指向性: 円
- 単一指向性: 前方に広いハート形
- スーパーカーディオイド: 前方が細く、後方にも小さい拾い

注記:
- 正確なポーラーパターンではなく概念図でよい

## mixer-channel-strip

目的: ミキサー1chの流れ。

図:
```text
入力端子 → ゲイン → EQ → AUX → フェーダー → MAIN
```

補足:
- ゲイン=入口
- フェーダー=ミックス上の音量
- AUX=別送り

## aux-routing-map

目的: MAINとAUXの違い。

図:
```text
ミキサー
├→ MAIN L/R: 講堂メイン
├→ AUX1: 録音PC
├→ AUX2: 通訳機
├→ AUX3: 通訳室
└→ AUX4: 控室
```

## feedback-loop

目的: ハウリングのループ。

図:
```text
マイク → ミキサー → アンプ → スピーカー
  ↑                              ↓
  └──── スピーカー音を再び拾う ────┘
```

デザイン:
- ループ矢印を赤系で表現
- 「ピー」「キーン」と小さく表示してもよい

## amp-power-calculation

目的: アンプ出力概算の入力要素。

図:
```text
必要音量 = 目標SPL + 距離 + スピーカー感度 + ヘッドルーム
```

正確には計算式ではなく、考慮要素として表示。

## cable-types

目的: ケーブル種類の整理。

図:
- XLR: マイク/バランス
- フォン: ライン/楽器
- RCA: 民生機器
- スピコン: パッシブスピーカー
- LAN: Dante
- HDMI/SDI: 映像系

## rehearsal-flow

目的: サウンドチェックの順番。

図:
```text
電源確認 → スピーカー確認 → マイク確認 → PC音源 → 録音 → 別室/通訳 → 本番状態へ
```

## live-operation-screen

目的: 本番中に見るポイント。

図:
- ミキサー画面風カード
- ONにするマイク
- ミュートするマイク
- 録音状態
- 次のBGM

## troubleshooting-flow

目的: 音が出ない時の切り分け。

図:
```text
入力は来ている？ → メーターは振れる？ → ミュート解除？ → 出力に送っている？ → スピーカー/アンプON？
```

## outdoor-distance-loss

目的: 屋外の音量差。

図:
- ステージ前方と後方の人
- 前方: 大きい
- 後方: 小さい
- 補助スピーカーを置く案を併記

## interpretation-routing

目的: 通訳・録音・別室送り。

図:
```text
ミキサー
├→ MAIN: 会場
├→ AUX1: 録音PC
├→ AUX2: 通訳機
├→ AUX3: 通訳室
└→ AUX4: 控室
```

注意:
- 通訳者音声が通訳者ヘッドホンへ戻らないよう、ループ注意の赤矢印を入れる。

## dante-basic

目的: Danteの基本構成。

図:
```text
Dante対応ミキサー → スイッチングハブ → Dante I/O / 録音PC / 別室
```

補足:
- LANケーブル
- CAT5e以上
- ギガビット対応ハブ
